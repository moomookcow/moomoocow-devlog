# 블로그 개선 개발 명세서

작성일: 2026-05-27  
상태: Draft v1

## 1. 문서 목적
- `README.md`와 분리하여, 실제 개선 개발에 필요한 요구사항/우선순위/완료기준을 관리한다.
- 구현 전에 범위를 고정하고, 개발 중 의사결정을 빠르게 하기 위한 기준 문서로 사용한다.

## 2. 이번 개선의 목표
- 운영 안정성 강화: 로그인/인가/세션 흐름을 실사용 수준으로 안정화
- 유입 기반 강화: SEO 기본 세트(메타데이터, sitemap, robots) 적용
- 품질 강화: 최소 테스트 자동화와 배포 체크리스트 정비
- 커뮤니티 안전성: 댓글 스팸 방어 및 모더레이션 기반 마련
- 모바일 완성도 강화: 모바일 반응형 깨짐/가독성/터치 UX 개선
- 체감 성능 강화: 재방문(이미 조회한 글) 시 로딩 지연 최소화
- 운영 기능 보강: 게시글 삭제 기능 추가

## 3. 범위
### In Scope
- 인증/인가 안정화
- SEO 개선
- 댓글 안전장치
- 테스트/배포 체크리스트
- 성능 기본 점검(병목 쿼리/불필요한 데이터 로딩 정리)
- 모바일 반응형 보수
- 게시글 삭제 기능(관리자)

### Out of Scope (이번 사이클 제외)
- 완전한 추천 시스템
- 대규모 디자인 리브랜딩
- 다국어(i18n) 지원
- 외부 검색엔진(Algolia 등) 연동

## 4. 우선순위 백로그
## P0
- 관리자 인증 플로우 안정화
  - `/admin/login -> /admin -> /admin/new` 정상 흐름 검증
  - 비인가 계정 접근 차단/리다이렉트 일관성 보장
- 모바일 반응형 이슈 수정
  - 주요 페이지(홈/상세/관리자)에서 모바일 레이아웃 깨짐/오버플로우 수정
  - 터치 타겟 및 가독성(폰트/간격) 점검
- SEO 기본 세트
  - 전역/페이지별 metadata 정리
  - `sitemap.ts`, `robots.ts` 추가
  - canonical URL 전략 정의
- 댓글 스팸 방어 1차
  - rate limiting(최소 수준)
  - 링크 과다/비정상 입력 차단 규칙

## P1
- 재방문 캐시/로딩 성능 개선
  - 이미 조회한 게시글 재진입 시 체감 로딩 감소
  - 라우트 prefetch/cache/revalidate 전략 점검 및 정리
- 게시글 삭제 기능
  - 관리자에서 게시글 삭제 액션 제공
  - 삭제 후 목록/상세/slug alias/연관 데이터 정합성 검증
- 테스트 자동화
  - lint, typecheck, 핵심 사용자 흐름 테스트(로그인->작성->출간->조회)
- 운영 체크리스트 문서화
  - 환경변수, 배포 후 스모크 테스트 절차
- 데이터 조회 최적화
  - 과도한 목록 조회/중복 조회 구간 정리

## P2
- 검색 고도화(서버 검색 또는 인덱스 전략)
- 댓글 모더레이션 UI(숨김/복원/신고 대응)
- 에디터 UX(저장 상태 가시화, 자동저장 등)
- 구글 서치 콘솔 연동 및 인덱싱 점검(최종 단계)

## 5. 작업 단위(에픽)
## Epic A. 인증/인가 안정화
- 요구사항
  - 인증 실패/세션 만료/비인가 시 일관된 리다이렉트
  - Admin Allowlist 검증 신뢰성 강화
- 완료기준
  - 실패 케이스별 기대 URL/메시지 명세 충족
  - 수동 테스트 체크리스트 전부 통과

## Epic B. SEO 기본 세트
- 요구사항
  - 홈/태그/포스트 상세 메타데이터 분리
  - 소셜 공유 메타(OG/Twitter) 반영
  - sitemap, robots 제공
- 완료기준
  - 주요 페이지 메타 누락 없음
  - sitemap/robots 라우트 동작 확인

## Epic C. 댓글 안전장치
- 요구사항
  - 과도한 요청 방어
  - 기본 유효성/스팸 규칙 강화
- 완료기준
  - 정상 댓글 작성은 유지
  - 반복/비정상 요청 차단 동작 확인

## Epic D. 품질 자동화
- 요구사항
  - `pnpm lint`/`pnpm exec tsc --noEmit`/핵심 흐름 테스트 자동 실행
- 완료기준
  - CI 또는 로컬 표준 커맨드로 재현 가능
  - 실패 시 원인 파악 가능한 로그 보장

## Epic E. 모바일 반응형 개선
- 요구사항
  - 모바일 뷰포트에서 깨짐/가로 스크롤/겹침 이슈 제거
  - 주요 인터랙션(태그 필터, 카드 클릭, 댓글 입력) 터치 UX 보장
- 완료기준
  - 홈(`/`), 태그(`/tags/[slug]`), 상세(`/posts/[slug]`), 관리자 핵심 화면 모바일 점검 통과
  - 수동 점검 체크리스트 항목 전부 통과

## Epic F. 게시글 캐시/재방문 성능 개선
- 요구사항
  - 이미 방문한 게시글 재진입 시 불필요한 재요청 최소화
  - prefetch 및 캐시 정책 명확화
- 완료기준
  - 재방문 시 로딩 시간 체감 개선 확인
  - 캐시 정책 문서화(어떤 페이지를 얼마 동안 캐시하는지)

## Epic G. 게시글 삭제 기능
- 요구사항
  - 관리자에서 게시글 삭제 수행 가능
  - 삭제 확인(Confirm) 단계 제공
  - 연관 데이터 처리 규칙 명확화(댓글/조회수/slug alias 등)
- 완료기준
  - 삭제 후 목록/상세 접근 결과가 기대대로 동작
  - 실수 방지를 위한 UX(경고/확인) 적용

## Epic H. 구글 서치 콘솔 연동(최종)
- 요구사항
  - 서비스 주요 개발 항목 완료 후 마지막 단계에서 진행
  - 소유권 인증(HTML 태그 또는 DNS) 및 사이트맵 제출
  - 인덱싱 상태/커버리지/모바일 사용성 점검
- 완료기준
  - Search Console 속성 인증 완료
  - `sitemap` 제출 완료 및 읽기 성공 확인
  - 핵심 페이지 인덱싱 요청 및 상태 확인

## Epic I. PDF 기반 초안 생성기(관리자)
- 요구사항
  - `/admin` 액션 버튼 영역에 `PDF 초안 생성` 버튼 추가
  - PDF 업로드 후 텍스트/코드 추출 및 블로그 초안 생성
  - OCR 과분리 텍스트(한글 글자 단위 띄어쓰기) 후처리 복원
  - 생성 결과를 `admin/new` 편집 플로우로 전달
- 완료기준
  - PDF 업로드 -> 초안 생성 -> 에디터 전달 1회 흐름 성공
  - 한글 OCR 과분리 케이스 복원 정확도 기준 충족
  - 텍스트층 PDF/스캔 PDF/코드 중심 PDF 샘플 검증 통과

### 제안 항목
- 기능명: PDF 기반 초안 생성기(관리자)
- 배경/문제:
  - 개발 서적을 PDF로 읽으며 스크린샷/코드/문장 단위로 정리하는 데 시간이 많이 든다.
  - 블로그 초안 작성의 시작 비용(구조화/요약/문장 정리)을 줄이고 싶다.
- 기대효과:
  - 독서 노트를 빠르게 게시글 초안 형태로 변환
  - 글 작성 속도 향상 및 발행 빈도 증가
  - 초안 품질의 일관성 확보(요약 구조/섹션 템플릿)
- 사용자 시나리오:
  - `/admin` 상단 액션 버튼 영역(카테고리 관리, 새 글 작성 옆)에 `PDF 초안 생성` 버튼 추가
  - 버튼 클릭 → PDF 업로드/입력 화면 이동
  - PDF에서 텍스트/코드 블록 추출 → 핵심 요약/목차/코드 설명 포함 초안 생성
  - 생성된 초안을 에디터(`admin/new`)로 전달하여 수정 후 저장/발행
- 범위(In/Out):
  - In: PDF 업로드, 텍스트/코드 추출, 요약 초안 생성, 관리자 에디터 연동
  - Out: 저작권 판단 자동화, 원문 대량 인용 자동 허용, 완전 자동 발행
- 기술 메모:
  - UI: `/admin` 대시보드 액션 버튼에 `PDF 초안 생성` 추가
  - Route: 예) `/admin/draft-from-pdf`
  - 처리:
    - PDF 파싱(텍스트 레이어 우선, 실패 시 OCR 전략은 추후 단계)
    - OCR 후처리(한글 정규화):
      - 글자 단위 과분리 교정(예: `안 녕 하 세 요` -> `안녕하세요`)
      - 불필요 공백/개행 정리, 문장부호 결합 복원
      - 코드 블록/일반 문단 분리 규칙 적용(코드는 공백 복원 대상에서 제외)
